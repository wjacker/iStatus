# Repository Rules

## Git Commit Messages

Use Conventional Commit style for every commit:

`type(scope): summary`

Rules:

- Always include a `type`
- Always include a `scope` in parentheses
- Keep the summary short and action-oriented
- Use lowercase for `type`, `scope`, and summary unless a proper noun requires capitalization

Examples:

- `chore(config): normalize project config`
- `feat(ui): add battery panel`
- `fix(metrics): avoid stale network samples`
- `docs(readme): add simplified chinese version`

This rule applies to all new threads and all future commits in this repository.
